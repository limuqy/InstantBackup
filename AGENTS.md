# AGENTS.md

## What this is

Multi-loader Minecraft server backup mod (Fabric / Forge / NeoForge). Incremental blob-based backups with XXHash64 dedup, async ZSTD compression, and a stop-server CLI.

- **Mod ID:** `instantbackup`
- **Package root:** `io.github.limuqy.mc.backup`
- **Primary language:** Chinese (code comments, log messages, docs)

## Build commands

```bash
# Build all loaders for a specific MC version (REQUIRED - no default works without -Pmc_ver)
./gradlew build -Pmc_ver=1.20.1

# Single loader
./gradlew :fabric:build -Pmc_ver=1.20.1

# Standalone CLI JAR (no MC dependency)
./gradlew :cli:shadowJar
# Output: cli/build/libs/instantbackup-cli-1.0.0-all.jar

# Dev server (auto-injects -Dinstantbackup.selftest=true)
./gradlew :fabric:runServer -Pmc_ver=1.20.1

# Build ALL versions (auto-selects correct JDK per version, Windows PowerShell)
powershell -File scripts/build-all.ps1
powershell -File scripts/build-all.ps1 -Versions 1.20.1,26.2  # specific versions
powershell -File scripts/build-all.ps1 -Clean                  # clean before build
```

Valid `mc_ver` values: `1.18.2`, `1.19.4`, `1.20.1`, `1.20.6`, `1.21.4`, `1.21.11`, `26.2`

Which loaders build for each version is controlled by `builds_for` in `versionProperties/<version>.properties`. Not all loaders exist for all versions (e.g. 1.18.2 has no NeoForge).

Requires JDK 25+ (to compile for 26.2's Java 25 target). Gradle Wrapper is included.

## Module structure

| Module | Role | MC dependency |
|--------|------|:-------------:|
| `core/` | Platform-agnostic backup engine (SQLite, ZSTD, blob store, config, i18n) | No |
| `common/` | MC integration: commands, mixins, scheduler, compat layer | Yes |
| `fabric/` | Fabric entrypoint + `FabricLoaderHelper` | Yes |
| `forge/` | Forge entrypoint + `ForgeLoaderHelper` | Yes |
| `neoforge/` | NeoForge entrypoint + `NeoForgeLoaderHelper` | Yes |
| `cli/` | Offline stop-server CLI (uses `core/` only) | No |
| `buildSrc/` | Gradle convention plugins (Loom, Manifold, Forgix) | ÔÇö |

Key rule: `core/` must never import Minecraft classes. `common/` must never import Fabric/Forge/NeoForge classes ÔÇö use `LoaderHelper` interface instead.

## Architecture patterns

**LoaderHelper abstraction:** Platform differences go through `LoaderHelper` interface (implemented by `FabricLoaderHelper`, `ForgeLoaderHelper`, `NeoForgeLoaderHelper`). Command registration is delegated to `CommandRegistry.register(dispatcher)`.

**Singleton entry:** `ExampleMod` holds the server instance, paths, and `LoaderHelper`. Accessed via `ExampleMod.mod()`. It is null before `SERVER_STARTING` ÔÇö Mixins and early code must handle this.

**Mixin rules:**
- All `@Inject` callbacks must be wrapped in try/catch ÔÇö mixins must never crash the game
- Use `catch (Throwable)`, not `catch (Exception)`
- New mixins must be registered in `common/src/main/resources/instantbackup.mixins.json`

**Compat layer** (`common/.../compat/`): Wraps MC API differences across versions (`ChatCompat`, `CommandCompat`, `PermissionCompat`, `ServerCompat`, `ModLog`, `CarpetCompat`). Check here first when adding MC API calls.

## Multi-version code (Manifold preprocessor)

Version branching uses Manifold `#if` directives with constants from `build.properties`:

```java
#if MC_VER <= MC_1_18_2
    // old API
#else
    // new API
#endif
```

Constants are generated by `root.gradle` ÔÇö e.g. `MC_1_20_1=2`, `MC_VER=2` when building with `-Pmc_ver=1.20.1`.

Rules:
- `#if` blocks are for API signature differences only, NOT business logic
- Version-specific MC API differences belong in `compat/` classes
- `build.properties` is auto-generated and gitignored ÔÇö never edit manually

## Build system details

**Gradle plugin chain:** `root.gradle` Ôćĺ `common.gradle` (Java + Manifold) Ôćĺ `minecraft.gradle` (Loom + MojMap) Ôćĺ loader-specific gradle scripts.

**Resource expansion:** `common.gradle` expands `${version}`, `${mod_id}`, `${java_version}`, etc. in `pack.mcmeta`, `fabric.mod.json`, `META-INF/mods.toml`, `META-INF/neoforge.mods.toml`, and `*.mixins.json`.

**Loader modules compile common/ source directly** ÔÇö `minecraft.gradle` adds `common/` and `core/` source sets to the loader's `compileJava` task, not just as jar dependencies. This means loader jars include all common code.

**Forgix** merges multi-loader outputs into a single JAR per MC version when `builds_for` lists more than one loader.

## Testing

```bash
# Start dev server ÔÇö look for [SelfTest] PASS in logs
./gradlew :fabric:runServer -Pmc_ver=1.20.1
# Check: fabric/run/server/logs/latest.log

# RCON-based full integration test (Windows PowerShell)
powershell -File scripts/verify_backup_rcon.ps1 -Loader fabric

# Wait for SelfTest result in a log file
powershell -File scripts/wait_selftest.ps1 -LogFile <path> -TimeoutSec 600
```

`runServer` injects `-Dinstantbackup.selftest=true` automatically. `BackupSelfTest` runs on `SERVER_STARTED`: create Ôćĺ wait idle Ôćĺ list Ôćĺ migrate Ôćĺ verify directory structure.

## File conventions

- Blob naming: `data/<relative-path>.<xxhash64>.zst`
- Config: `config/instantbackup/instantbackup.properties`
- Metadata: `config/instantbackup/metadata/` (CSV, default) or `backups.db` (SQLite) or MySQL remote ÔÇö see `docs/metadata-storage.md`
- Backups: `backups/` (configurable via `storage.path`, supports absolute cross-drive paths)

## Common pitfalls

1. **`ExampleMod.mod()` is null before SERVER_STARTING** ÔÇö Mixin callbacks and early init code must guard against this
2. **Windows `session.lock`** ÔÇö may be locked by running server; scanning must skip, not fail
3. **Don't import loader APIs in `common/`** ÔÇö add to `LoaderHelper` interface instead
4. **Don't put business logic in `#if` blocks** ÔÇö use `compat/` classes
5. **Resource files use `${}` expansion** ÔÇö don't hardcode version strings in mod metadata files
6. **`build.properties` is gitignored and auto-generated** ÔÇö changing MC version requires rebuild, not manual edit
